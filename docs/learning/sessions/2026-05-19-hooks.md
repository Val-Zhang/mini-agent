# Session: Day 11 Hooks (2026-05-19)

## Goal

实现一个最小 Hook 系统，让 agent run 和工具执行生命周期可以被外部逻辑观察或拦截。

## Core Concepts

- **SessionStart**：一次 `AgentRunner.run()` 接收用户输入并写入 history 后触发。
- **SessionEnd**：run 以 completed、failed 或 cancelled 收尾时触发。
- **PreToolUse**：权限通过后、真实工具执行前触发；可以返回 block 阻止工具执行。
- **PostToolUse**：工具调用形成最终 tool result 后触发；包括真实执行、permission deny、用户拒绝和 hook block。

## Implementation

1. 新增 `src/agent/hooks/`：
   - `types.ts`：定义 hook 生命周期上下文、失败策略和 block 结果。
   - `HookRegistry.ts`：按顺序调用 hook，支持 pre-tool block。
   - `auditHook.ts`：把 session 和工具执行记录追加到 `memories/session/audit.jsonl`。
   - `defaultHooks.ts`：默认注册 audit hook。
2. `AgentRunner` 接入生命周期：
   - `run()` 里触发 session start/end。
   - `executeToolCall()` 里在 permission 之后触发 pre hook。
   - 所有 tool result 都触发 post hook，包括被权限或 hook 拦截的调用。
3. `createAgent()` 默认安装 audit hook，手写测试用 `AgentRunner` 时默认空 hook registry，避免测试产生文件副作用。

## Audit Record

默认审计文件：

```text
memories/session/audit.jsonl
```

目前记录三类事件：

- `session_start`：input、mode。
- `session_end`：input、mode、status、message。
- `tool_use`：tool name、input preview、permission decision/reason、duration、isError、output preview。

## Failure Policy

- Hook 默认 `warn`：hook 抛错时打印警告，不中断主流程。
- `failurePolicy: "block"`：hook 抛错时阻止流程。
- `onPreToolUse` 可以显式返回 `{ action: "block", reason }` 阻止工具。

## Tests

- `test/hooks.test.ts` 覆盖 registry 调用顺序、pre hook block、audit hook 持久化。
- `test/agent-runner.test.ts` 覆盖 runner 中 session/tool hook 顺序和 pre hook 阻止真实执行。
- `test/create-agent.test.ts` 覆盖默认 agent 会安装 audit hook。

## Trade-offs

- 当前没有把 hook 作为 CLI 可配置项暴露；默认只开启审计 hook。
- 审计记录使用 preview 截断，不做敏感信息脱敏；后续 Product Hardening 需要补脱敏策略。
- Hook 事件暂不进入 terminal renderer，先保持为内部扩展点。
