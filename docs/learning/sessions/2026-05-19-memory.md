# Session: Day 12 Memory (2026-05-19)

## Goal

实现一个显式、可审计、可读写的长期 memory 最小闭环，并把它和 transcript、compact summary、audit log 分开。

## Design Principles

- **Memory is not transcript**：transcript 是完整对话事实；memory 是被筛选后未来有用的长期知识。
- **Memory is not compact**：compact 解决当前长会话继续运行；memory 跨会话复用。
- **Explicit write first**：第一版不自动从 transcript 抽取记忆，只通过 `remember` 工具显式写入。
- **Small summary injection**：模型调用前只注入精简 memory summary，不把所有日志塞进上下文。
- **Writes need permission**：`remember` 会长期影响后续行为，所以默认走 permission `ask`。

## Memory Scopes

当前支持三类长期记忆：

- `project_fact`：稳定项目事实，写入 `memories/project/facts.md`。
- `user_preference`：用户长期偏好，写入 `memories/user/preferences.md`。
- `workflow_note`：工作流/实践经验，写入 `memories/project/workflow.md`。

## Implementation

新增 `src/agent/memory/`：

- `types.ts`：memory scope、entry、snapshot。
- `memoryStore.ts`：markdown 文件读写。
- `memoryManager.ts`：内容归一化、去重、summary 格式化。

新增 `src/tools/memory/`：

- `schema.ts`：`remember` tool schema。
- `rememberTool.ts`：显式保存长期记忆。

接入点：

1. `createDefaultTools()` 注册 `remember`。
2. `PermissionManager` 把 `remember` 作为写入类工具，默认 `ask`。
3. `AgentRunner` 持有 `MemoryManager`。
4. `ContextManager` 在 compact summary 之后注入：

```text
Long-term memory:

## Project Facts
- ...

## User Preferences
- ...

## Workflow Notes
- ...
```

## Files and Responsibilities

- `memories/session/transcript.jsonl`：完整对话事实。
- `memories/session/compact.md`：当前会话压缩摘要。
- `memories/session/audit.jsonl`：session/tool 审计记录。
- `memories/session/plan.md`：当前计划状态。
- `memories/project/facts.md`：项目长期事实。
- `memories/project/workflow.md`：项目工作流经验。
- `memories/user/preferences.md`：用户长期偏好。

## Tests

- `test/memory.test.ts`：memory markdown 持久化、去重、summary、remember tool。
- `test/context.test.ts`：long-term memory 和 compact summary 分别注入。
- `test/agent-runner.test.ts`：runner 会把 long-term memory 送入模型上下文。
- `test/permissions.test.ts`：`remember` 默认需要确认。
- `test/tools.test.ts`：默认工具集合包含 `remember`。

## Trade-offs

- 当前不做 embedding、向量检索或图记忆。
- 当前不做自动 memory extraction，避免错误长期污染。
- 当前 markdown 解析只支持 `- memory _(saved timestamp)_` 这种简单格式。
- 当前 memory summary 全量读取每个 scope 的最后若干条，后续可以升级为索引和按需召回。

## Next Step

- 给 `remember` 增加更细的用户确认文案和 renderer 展示。
- 后续可加入 memory review 命令，允许用户列出、编辑、删除长期记忆。
- 等记忆增长后，再探索 OpenAI Agents SDK 风格的 progressive disclosure：先注入 summary，再按需检索详细 memory。
