# Session: Day 8 Skills (2026-05-05)

## 今日目标

- 引入 `skills/*/SKILL.md` 约定。
- 提供按需加载能力（`load_skill`），而不是把所有技能直接注入主系统提示词。
- 保持 plan mode 可用（允许读取技能说明，但不放开写操作）。

## 实践前的最佳实践调研（摘要）

我们先参考了社区主流实践（如 Claude Code Skills 文档）并提炼出三条对当前项目最关键的原则：

1. 技能应“按需加载”，避免系统提示词常驻膨胀。
2. 技能文件应“结构化元数据 + 自由正文指令”分层管理。
3. 运行时要先暴露“技能列表与描述”，模型再按需请求正文，形成可观测的调用链。

## 设计决策

1. 文件约定：`skills/<skill-name>/SKILL.md`
2. Frontmatter 字段最小化：`name`、`description`
3. 正文作为技能指令主体，不做二次改写
4. 新增 `load_skill` 工具：
   - `action: list` 返回可用技能摘要
   - `action: load` 返回指定技能正文
5. `plan` 模式允许 `load_skill`（只读行为）

## 代码改动

- `src/agent/skills/*`：技能 registry、frontmatter 解析、loader
- `src/tools/skills/*`：`load_skill` 工具与 schema
- `src/tools/defaultTools.ts`：默认工具集注入 `load_skill`
- `src/agent/createAgent.ts`：启动时加载技能 registry
- `src/agent/mode/policy.ts`：plan mode allowlist 增加 `load_skill`
- `skills/researcher/SKILL.md`、`skills/reviewer/SKILL.md`：示例技能
- `test/skills.test.ts`：技能解析、加载、工具行为测试
